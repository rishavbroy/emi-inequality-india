## Methodological Contract Coverage

The remaining work should not be driven by numeric parity alone. The refactor must preserve the following methodological contracts unless a row in this ledger explicitly classifies the change as an `accepted_deviation`.

### A. Selection/probit contract

The active probit specification is intended to retain the legacy design. The child-level selection sample is built from NSS 2007 education Block 5 enrolled-child records, Block 6 enrolled-child expenditure records, and Block 4 demographics for all 5--19 year-olds. The active builder preserves the legacy full-join behavior and explicit reference categories for religion, social group, primary-school distance, and father education. The probit covariate set remains:

- controls: `AGE`, `SEX`, `HH_SIZE`, `RELIGION`, `SOCIAL_GROUP`, `SECTOR`;
- all-child exclusions: `DIST_FROM_NEAREST_PRIMARY_CLASS`, `father_educ`;
- enrolled-child district aggregates: `dmean_num_IS_EDU_FREE`, `dmean_num_TUTION_FEE_WAIVED`, `dmean_num_RECD_SCHOLARSHIP_STIPEND`, `dmean_num_RECD_TXT_BOOKS`, `dmean_num_RECD_STATIONERY`, `dmean_num_MID_DAY_MEAL_ETC_RECD`, `dmean_num_ENROLLMENT_COST`.

The final-mode selection model continues to use a survey-weighted probit with PSU `FSU_SL_NO`, weights from `weight`, strata built from `STATE`, `STRATUM`, and `SUB_STRATUM_NO`, nested sampling, and `survey.lonely.psu = "average"`. The full-household father-proxy key is an accepted correction to the legacy `HHID`-only join because `HHID` alone is not globally unique after raw-file standardization.

### B. District pseudo-panel contract

The district pseudo-panel must preserve the legacy research design: build a district tracker across 2001, 2007--08, 2017--18, and 2020; attach 2001 linguistic distance, 2007--08 baseline/treatment controls, 2017--18 outcomes, and 2020 geometry; then restrict the IV analysis panel to rows with the core treatment, instrument, baseline consumption, and follow-up consumption available.

The active refactor intentionally differs from the legacy implementation in one documented way: source-to-tracker matching keeps a deterministic one-source-to-one-tracker safeguard. This avoids silently duplicating district-level measures through many-to-many fuzzy candidates. The safeguard is a reproducibility correction; it is not a license to accept the current IV sample without further audit. The current `N = 482` panel is accepted as a corrected analysis sample because it preserves the tracker/source/methodological contract while avoiding implicit many-to-many source duplication and legacy matching losses. Table 4/5/6 legacy numeric differences should remain visible as warnings and comparison context.

### C. Measure-construction contract

The active measure builders are intended to retain the legacy block-specific construction:

- `EMIE`: from NSS 2007 education Block 5, children with `AGE <= 19`, weighted mean of `MEDIUM_INSTRUCTION == "02"`, scaled to percent.
- 2007 population/households/consumption/Gini: from NSS 2007 education Block 3, one row per district-household, household-level weight, per-capita consumption `TOTAL / HH_SIZE`, and weighted Gini.
- 2007 demographic controls: from NSS 2007 education Block 4, weighted district shares/ratios for urban status, dependency ratio, female-headed households, religion, social group, land, and head education.
- 2007 housing control: from NSS 2007 consumption household characteristics, weighted district share of pucca structures.
- 2017 consumption/Gini: from NSS 2017 education data using district code `NSS_Region + District`, `MULT_Combined` weights, and per-capita consumption when household consumption and household size are provided.
- 2001 linguistic distance: population-weighted average of Hindi-distance scores over Census 2001 mother-tongue speaker totals by district.

The new `test-methodology-contract.R` tests protect these methodological choices. Passing these tests should be read as evidence that the documented design is retained; it does not by itself settle the unresolved IV panel row set or coefficient differences.

### 2026-06-11 district-source fuzzy cascade refinement

The district-panel builder now applies the legacy source-attachment cascade more literally: after choosing the nearest tracker suffix for each source year, unmatched source rows are matched within state using the legacy method/threshold sequence `soundex == 0`, `qgram == 0`, `jw <= 0.15`, `dl <= 2`, and `osa <= 1`. The refactor still keeps the deliberate one-source-to-one-tracker safeguard to avoid silently duplicating source measures across tracker rows; this is a reproducibility correction to the legacy `stringdist_join(..., mode = "full")` workflow, whose many-to-many candidates were later collapsed implicitly when columns were merged back by tracker row. Remaining IV-panel mismatches should therefore be interpreted as unresolved district-panel/measure-construction differences, not as license to force the legacy PDF numbers.
